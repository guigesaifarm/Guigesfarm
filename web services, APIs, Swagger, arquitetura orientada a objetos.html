Aqui estão 10 questões de web services, APIs, Swagger, arquitetura orientada a objetos e arquitetura de aplicações para ambientes web, no estilo da banca FGV, com as respostas explicadas.

1. (FGV - 2023)

Um web service pode ser descrito como:

a) Uma interface gráfica para a comunicação entre o usuário e a aplicação.
b) Um componente de hardware que conecta sistemas de diferentes plataformas.
c) Uma aplicação que expõe funcionalidades por meio de protocolos e padrões de comunicação, como HTTP e SOAP.
d) Uma API que exige a autenticação de usuários para acessar seus dados.

Resposta: c) Uma aplicação que expõe funcionalidades por meio de protocolos e padrões de comunicação, como HTTP e SOAP.

Explicação:
Um web service é uma aplicação que expõe funcionalidades por meio de protocolos de comunicação, como HTTP, SOAP ou REST, permitindo a integração entre sistemas em plataformas diferentes. Web services são projetados para permitir a interoperabilidade entre aplicações distribuídas.

2. (FGV - 2022)

Em relação a APIs (Application Programming Interface), podemos afirmar que:

a) APIs são utilizadas apenas para integrar sistemas que utilizam a mesma linguagem de programação.
b) APIs permitem a comunicação entre sistemas e aplicações de maneira padronizada, independentemente da linguagem ou plataforma.
c) APIs são exclusivamente utilizadas para interação com banco de dados.
d) APIs não precisam de documentação para funcionar corretamente.

Resposta: b) APIs permitem a comunicação entre sistemas e aplicações de maneira padronizada, independentemente da linguagem ou plataforma.

Explicação:
APIs são interfaces padronizadas que permitem a comunicação entre sistemas e aplicações de diferentes linguagens e plataformas. Elas são independentes de linguagem e plataforma, facilitando a integração entre sistemas diversos, utilizando protocolos como HTTP, SOAP, REST e formatos de dados como JSON e XML. A documentação da API, embora útil, não é obrigatória para o funcionamento, mas é essencial para o uso correto.

3. (FGV - 2021)

Em um contexto de Swagger (ou OpenAPI), o principal benefício dessa ferramenta é:

a) Automatizar a geração do código do servidor e cliente de uma API.
b) Criar interfaces gráficas para a comunicação com bancos de dados.
c) Garantir a segurança da comunicação em APIs RESTful.
d) Substituir a necessidade de especificações formais em APIs.

Resposta: a) Automatizar a geração do código do servidor e cliente de uma API.

Explicação:
Swagger (agora chamado de OpenAPI) é uma ferramenta que permite a documentação, geração de código e testes automatizados para APIs RESTful. Ele facilita a criação da especificação da API, permitindo que tanto o código do servidor quanto do cliente sejam gerados automaticamente a partir de uma descrição da API em formato OpenAPI (antes conhecido como Swagger). Isso simplifica a integração entre sistemas e reduz o esforço de desenvolvimento.

4. (FGV - 2020)

Qual das alternativas abaixo é uma característica fundamental da arquitetura orientada a objetos?

a) Acessibilidade de funções apenas a partir de interfaces gráficas.
b) Uso de classes e objetos para modelar entidades do mundo real.
c) Centralização do controle de todos os objetos em um único componente.
d) Armazenamento de todas as informações em bancos de dados relacionais.

Resposta: b) Uso de classes e objetos para modelar entidades do mundo real.

Explicação:
A arquitetura orientada a objetos é baseada na ideia de modelar entidades do mundo real como objetos, que possuem atributos (propriedades) e métodos (funcionalidades). A principal característica de sistemas orientados a objetos é a abstração, que permite representar fenômenos complexos de maneira mais natural, utilizando classes para definir tipos de objetos e seus comportamentos.

5. (FGV - 2023)

A arquitetura de aplicações para ambiente web deve ser projetada para garantir:

a) A implementação de sistemas monolíticos com todas as funcionalidades interligadas.
b) A comunicação entre o servidor e o cliente por meio de APIs bem definidas e escalabilidade.
c) A utilização de um único banco de dados para todas as funcionalidades da aplicação.
d) A utilização de uma única tecnologia de front-end para garantir uniformidade visual.

Resposta: b) A comunicação entre o servidor e o cliente por meio de APIs bem definidas e escalabilidade.

Explicação:
Arquiteturas de aplicações para ambientes web devem garantir uma comunicação eficiente entre o cliente (geralmente o navegador) e o servidor, utilizando APIs bem definidas (geralmente RESTful ou GraphQL). Além disso, a escalabilidade é essencial, pois é necessário que a aplicação consiga lidar com um número crescente de usuários ou de dados de forma eficiente, o que exige o uso de técnicas como balanceamento de carga, microservices, entre outros.

6. (FGV - 2022)

A principal diferença entre arquitetura monolítica e arquitetura de microservices é:

a) Na arquitetura monolítica, os componentes são desenvolvidos e implantados como uma única unidade; já na arquitetura de microservices, os componentes são independentes e podem ser implantados separadamente.
b) A arquitetura de microservices centraliza todas as funcionalidades em um único servidor, enquanto a arquitetura monolítica as distribui entre diferentes servidores.
c) A arquitetura monolítica utiliza tecnologias web, enquanto a arquitetura de microservices utiliza apenas tecnologias desktop.
d) Na arquitetura monolítica, a aplicação é desacoplada, enquanto na arquitetura de microservices todos os componentes estão fortemente acoplados.

Resposta: a) Na arquitetura monolítica, os componentes são desenvolvidos e implantados como uma única unidade; já na arquitetura de microservices, os componentes são independentes e podem ser implantados separadamente.

Explicação:
Na arquitetura monolítica, todos os componentes (front-end, back-end, banco de dados) são desenvolvidos e implantados juntos como uma única unidade. Já na arquitetura de microservices, a aplicação é dividida em serviços independentes, que podem ser desenvolvidos, testados e implantados separadamente. Essa abordagem facilita a escalabilidade e a manutenção do sistema, pois mudanças em um serviço não afetam diretamente os outros.

7. (FGV - 2021)

Qual é o principal objetivo da arquitetura RESTful em relação à construção de APIs para aplicações web?

a) Permitir que a API funcione apenas em ambientes corporativos internos.
b) Simplificar a comunicação entre cliente e servidor por meio de métodos HTTP como GET, POST, PUT e DELETE.
c) Garantir que a API tenha uma interface gráfica para o usuário final.
d) Limitar o número de operações que o servidor pode realizar por requisição.

Resposta: b) Simplificar a comunicação entre cliente e servidor por meio de métodos HTTP como GET, POST, PUT e DELETE.

Explicação:
A arquitetura RESTful define uma série de convenções e princípios para construir APIs simples e eficientes. Ela utiliza os métodos HTTP padrão (GET, POST, PUT, DELETE) para realizar operações CRUD (Create, Read, Update, Delete) sobre recursos. Essa abordagem sem estado e baseada em recursos simplifica a comunicação entre cliente e servidor, facilitando a escalabilidade e manutenção.

8. (FGV - 2020)

Em relação à arquitetura de software em aplicações web, uma das vantagens da arquitetura orientada a objetos é:

a) A facilidade de criar interfaces de usuário complexas sem necessidade de lógica de negócio.
b) A possibilidade de modelar entidades e suas interações de maneira modular, promovendo reutilização e manutenção.
c) A centralização de todos os dados em um único banco de dados relacional.
d) A implementação de todas as funcionalidades no lado do servidor, sem uso de JavaScript.

Resposta: b) A possibilidade de modelar entidades e suas interações de maneira modular, promovendo reutilização e manutenção.

Explicação:
A arquitetura orientada a objetos permite a modelagem de entidades e suas interações de forma modular, usando classes e objetos. Essa abordagem favorece a reutilização de código e facilita a manutenção da aplicação, já que mudanças em uma parte do sistema não afetam diretamente outras partes, desde que as interfaces e contratos entre os objetos permaneçam inalterados.

9. (FGV - 2022)

O Swagger (ou OpenAPI) pode ser utilizado para:

a) Controlar o tráfego de dados entre o servidor e o cliente.
b) Descrever e documentar a API, além de permitir testes e geração automática de código.
c) Criar um banco de dados centralizado para armazenar os dados da aplicação.
d) Implementar a segurança e criptografia das mensagens enviadas entre o cliente e o servidor.

Resposta: b) Descrever e documentar a API, além de permitir testes e geração automática de código.

Explicação:
Swagger (OpenAPI) é uma ferramenta que facilita a documentação e especificação de APIs RESTful. Ele permite criar uma documentação interativa para a API, testar as operações diretamente e até gerar o código do cliente e do servidor automaticamente. O Swagger não é responsável por controle de tráfego, segurança ou criptografia diretamente, mas facilita o desenvolvimento e a integração de APIs.

10. (FGV - 2021)

Em uma arquitetura de aplicações web utilizando microservices, uma das principais vantagens é:

a) A possibilidade de centralizar toda a lógica de negócio em um único serviço.
b) A facilidade de implementar alterações em um serviço sem impactar outros serviços da aplicação.
c) A utilização de um único banco de dados para todas as funcionalidades.
d) A complexidade de gerenciamento de várias instâncias de serviços.

Resposta: b) A facilidade de implementar alterações em um serviço sem impactar outros serviços da aplicação.

Explicação:
Na arquitetura de microservices, a aplicação é composta por vários serviços independentes. Isso significa que alterações em um serviço podem ser realizadas de forma isolada, sem afetar os outros serviços, o que facilita a manutenção e a escalabilidade da aplicação. A descentralização da lógica de negócio e do armazenamento de dados é uma das vantagens dessa abordagem.
