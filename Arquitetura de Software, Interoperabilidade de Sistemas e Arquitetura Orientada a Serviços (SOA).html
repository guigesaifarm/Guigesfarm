1. (FGV - 2022)

Em uma Arquitetura Orientada a Serviços (SOA), qual dos seguintes componentes é responsável por expor os serviços para os consumidores de forma padronizada?

a) Service Bus
b) Service Registry
c) Service Consumer
d) Service Provider

Resposta: b) Service Registry

Explicação:
O Service Registry é um componente fundamental em uma arquitetura SOA. Ele atua como um repositório onde os serviços são registrados, e os consumidores podem procurar e se conectar a eles de maneira padronizada. O Service Bus é usado para o roteamento e integração de mensagens entre os serviços, enquanto o Service Consumer é o cliente que utiliza os serviços, e o Service Provider é o serviço que oferece funcionalidades.

a) HTTP
b) SOAP
c) REST
d) SMTP

Resposta: b) SOAP

Explicação:

Ele é um protocolo baseado em XML e permite a troca de informações de maneira padronizada e segura entre sistemas heterogêneos. O REST também é utilizado em serviços, mas ele é mais simples e menos formal em comparação com o SOAP.

O SOAP (Simple Object Access Protocol) é um protocolo de comunicação que permite a troca de mensagens entre sistemas através da rede, usando o XML como formato de mensagem. Ele é amplamente utilizado em sistemas baseados em Arquitetura Orientada a Serviços (SOA) para permitir que diferentes sistemas, muitas vezes com plataformas diferentes, possam se comunicar de forma padronizada.

O SOAP é um protocolo independente de plataforma, o que significa que ele pode ser utilizado em sistemas diferentes (Windows, Linux, etc.) e com diferentes linguagens de programação (Java, .NET, Python, etc.). As mensagens SOAP são enviadas via protocolos de rede como HTTP ou SMTP e seguem uma estrutura muito bem definida.

Abaixo, apresento um exemplo simples de um algoritmo SOAP e um código para invocar e processar uma requisição SOAP. Vou fornecer um exemplo básico usando a linguagem Java com a biblioteca JAX-WS (Java API for XML Web Services), que facilita a criação de serviços SOAP.

Exemplo 1: Serviço Web SOAP (Servidor)

Aqui está um exemplo simples de um servidor SOAP que oferece um serviço para somar dois números.

Código do Servidor SOAP em Java:

java
Copiar código
import javax.jws.WebService;
import javax.jws.WebMethod;

// Definindo o serviço SOAP
@WebService
public class CalculatorService {
    
    // Método que será exposto como um serviço SOAP
    @WebMethod
    public int add(int a, int b) {
        return a + b;
    }
    
    // Método para iniciar o serviço
    public static void main(String[] args) {
        try {
            // Publica o serviço na URL
            javax.xml.ws.Endpoint.publish("http://localhost:8080/calculator", new CalculatorService());
            System.out.println("Serviço SOAP publicado em http://localhost:8080/calculator");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Explicação:

  1. @WebService: Esta anotação define a classe CalculatorService como um serviço SOAP.

  2. @WebMethod: A anotação é usada para marcar o método add() como um método exposto via SOAP. Esse método vai somar dois números e retornar o resultado.

  3. Endpoint.publish(): A chamada Endpoint.publish() publica o serviço SOAP em uma URL (neste caso, http://localhost:8080/calculator).

Exemplo 2: Cliente SOAP (Cliente que consome o serviço)

Agora, vamos criar um cliente SOAP que consome o serviço que acabamos de definir.

Código do Cliente SOAP em Java:

java
Copiar código
import javax.xml.namespace.QName;
import javax.xml.ws.Service;
import java.net.URL;

public class CalculatorClient {
    public static void main(String[] args) {
        try {
            // URL do WSDL (Service Definition)
            URL url = new URL("http://localhost:8080/calculator?wsdl");
            
            // QName que define o namespace e o nome do serviço
            QName qname = new QName("http://localhost:8080/", "CalculatorService");
            
            // Criação do serviço SOAP
            Service service = Service.create(url, qname);
            
            // Acessando o método do serviço
            CalculatorService calculator = service.getPort(CalculatorService.class);
            
            // Chamando o método 'add' do serviço SOAP
            int result = calculator.add(5, 3);
            System.out.println("Resultado da soma: " + result);  // Saída esperada: 8
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Explicação:

  1. URL do WSDL: O cliente SOAP precisa do endereço do WSDL (Web Service Description Language), que é um documento XML que descreve como interagir com o serviço SOAP. O WSDL está disponível na URL do serviço (http://localhost:8080/calculator?wsdl).

  2. QName: O QName especifica o namespace e o nome do serviço que o cliente SOAP precisa acessar.

  3. Service.create(): Este método cria o serviço a partir da URL do WSDL e do QName.

  4. getPort(): O getPort() retorna a interface do serviço que define os métodos disponíveis. No nosso caso, ele mapeia a interface CalculatorService que expõe o método add().

  5. Chamada ao método: O cliente então chama o método add() e imprime o resultado.

Exemplo de Mensagem SOAP (XML)

Quando o cliente faz a requisição ao serviço, a mensagem SOAP enviada poderia ser algo assim (mensagem XML):

Conclusão

Neste exemplo, o código Java foi usado para criar um serviço web SOAP simples e um cliente SOAP que consome o serviço.

  • O servidor SOAP expõe um serviço de soma de dois números, enquanto o cliente SOAP envia a requisição e recebe a resposta.

  • As mensagens SOAP trocadas entre o cliente e o servidor são no formato XML, que é o formato padrão de comunicação do SOAP.

A principal vantagem do SOAP é que ele é um protocolo robusto e padronizado, ideal para integração de sistemas complexos, especialmente em ambientes corporativos, onde é necessário garantir a interoperabilidade entre sistemas heterogêneos.
